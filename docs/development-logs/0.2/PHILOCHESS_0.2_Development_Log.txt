Date: July 12, 2026

PHILOCHESS 0.2-alpha-001 — Basic quiescence search

Implemented the first basic quiescence search in PHILOCHESS.

Technical milestone:
PHILOCHESS can now continue searching tactically unstable positions after the 
normal alpha-beta depth reaches zero.

Instead of immediately applying the static material evaluation at depth zero, 
the engine now examines forcing tactical continuations until it reaches a 
position considered sufficiently quiet for static evaluation.

This is the first version of the PHILOCHESS 0.2 development cycle.
It builds directly on PHILOCHESS 0.1-alpha-032, the validated final version of 
PHILOCHESS 0.1.

Alpha-001 preserves:
- legal move generation;
- FEN loading;
- castling;
- en passant;
- pawn promotion;
- check, checkmate, and stalemate detection;
- perft and divide;
- known reference perft validation;
- material evaluation;
- basic alpha-beta search;
- basic UCI time-control parsing;
- tactical regression testing;
- basic move ordering.

New functionality:
- Added basic quiescence search.
- Alpha-beta now enters quiescence when the normal search depth reaches zero.
- Added stand pat evaluation when the side to move is not in check.
- Added stand pat alpha updates.
- Added stand pat beta cutoffs.
- Added recursive quiescence search using the negamax framework.
- Added captures to the quiescence move set.
- Added promotions to the quiescence move set, including non-capturing 
promotions.
- Deliberately excluded non-capturing checking moves from this first 
quiescence version.
- Added complete legal check-evasion search when the side to move is in check.
- Disabled stand pat when the side to move is in check.
- Added terminal checkmate and stalemate handling inside quiescence.
- Added periodic time-limit checks inside quiescence.
- Added an emergency quiescence safety limit of 64 plies.
- Added a counter for quiescence ply-limit hits.
- Added separate counters for:
  - main-search nodes;
  - quiescence nodes;
  - alpha-beta cutoffs;
  - quiescence beta cutoffs;
  - quiescence nodes entered while in check;
  - selective search depth;
  - quiescence ply-limit hits.
- Updated search information output to report the new quiescence statistics.

Move ordering refinement:
The MVV-LVA-style capture ordering introduced in PHILOCHESS 0.1-alpha-031 was 
reviewed and refined.

The original material values could treat the king as a zero-value attacker 
because the king has a material evaluation value of 0.
Alpha-001 now uses separate piece-ordering ranks for MVV-LVA:

Pawn   = 1
Knight = 2
Bishop = 3
Rook   = 4
Queen  = 5
King   = 6

This keeps material evaluation values separate from move-ordering ranks.

The refined ordering now ensures that:
- the most valuable victim receives priority;
- when two moves capture the same victim, the less valuable attacker receives 
priority;
- the king is no longer treated as the cheapest possible attacker;
- promotions receive their own high-priority category;
- capture-promotions receive both promotion and capture bonuses.

New internal diagnostic commands:
- qprobe
- qcompare [depth]
- qtest

qprobe:
Runs quiescence directly from the current position and reports:
- qscore;
- qnodes;
- quiescence beta cutoffs;
- quiescence check nodes;
- selective depth;
- ply-limit hits.

qcompare [depth]:
Runs the same search twice:
- with MVV-LVA ordering inside quiescence;
- without move ordering inside quiescence.

This allows direct comparison of the resulting move, evaluation, nodes, 
qnodes, and quiescence cutoffs.

qtest:
Runs four controlled internal tests for:
- horizon recapture avoidance;
- capture-sequence resolution;
- non-capturing promotion inclusion;
- check evasion without stand pat.

Current limitations:
- The static evaluation remains material-only.
- Quiescence currently searches captures and promotions only when the side to 
move is not in check.
- Non-capturing checking moves are deliberately excluded.
- No static exchange evaluation is implemented.
- No SEE pruning is implemented.
- No delta pruning is implemented.
- No transposition table is implemented.
- No hash move ordering is implemented.
- No principal variation move ordering is implemented.
- No killer move heuristic is implemented.
- No history heuristic is implemented.
- No iterative deepening is implemented.
- Maximum normal search depth remains 4.
- The principal variation output still reports only the selected root move.
- Time management remains basic.
- stop is recognized, but search is still synchronous and cannot yet be 
interrupted asynchronously.
- go infinite is parsed, but true infinite analysis is not implemented.
- Quiescence nodes can significantly outnumber normal search nodes in tactical 
positions.
- The 64-ply limit is an emergency safety mechanism, not a replacement for 
future selective pruning.
- The tactical regression suite is still small and should be expanded with 
additional controlled positions and future errors found in PHILOCHESS games.
- The engine does not yet evaluate piece placement, mobility, king safety, 
pawn structure, development, activity, coordination, or initiative.
- Draw adjudication by the fifty-move rule, repetition, or insufficient 
material is not implemented.
- The current Debug build is not optimized for speed.
- External Myracle tournament validation of alpha-001 remains pending.

New internal diagnostic commands:
- inventorytest [depth]
- fullregression [depth]

inventorytest [depth]:
Executes the complete 64-case inventory. If no depth is supplied, category [B]
search targets use diagnostic depth 4.

The command reports for every case:
- its stable identifier;
- category;
- result state;
- test title;
- the relevant legal, tactical, search, node, or evaluation data.

It concludes with:
- PASS, FAIL, OPEN, BASELINE, and TOTAL counts;
- category counts;
- an inventory structure result.

fullregression [depth]:
Provides an alias for the same complete inventory execution.

Meaning of result states:
- PASS means that the case fulfilled its implemented criterion.
- FAIL means that the test infrastructure or required criterion failed.
- OPEN means that the position and search executed correctly, but the 
documented
  tactical target was not demonstrated at the tested depth.
- BASELINE means that a category [C] position was loaded and verified as a
  reference without requiring positional understanding that the engine does not
  yet possess.

PHILOCHESS 0.2-alpha-003 — Iterative Deepening

Implemented the first iterative-deepening search in PHILOCHESS.

Technical milestone:
PHILOCHESS can now search successively from depth 1 to the requested depth,
preserve the result of the last fully completed iteration, and use the previous
iteration's best root move to guide the next iteration.

This is the third version of the PHILOCHESS 0.2 development cycle.
It builds directly on the validated PHILOCHESS 0.2-alpha-002 code without
modifying that closed version.

Alpha-003 preserves:
- legal move generation and move application;
- FEN loading;
- castling;
- en passant;
- pawn promotion;
- check, checkmate, and stalemate detection;
- perft, divide, and known reference perft validation;
- material evaluation;
- alpha-beta search;
- quiescence search;
- refined MVV-LVA move ordering;
- UCI mate-score output;
- basic UCI time-control parsing;
- the four-case quiescence correctness suite;
- the inherited three-position tactical regression test;
- the complete 64-case tactical regression and quiescence inventory;
- all diagnostic commands inherited from alpha-002.

New functionality:
- Replaced the normal single fixed-depth playing search with iterative
  deepening.
- Added successive completed searches at depths 1, 2, 3, and 4 when depth 4 is
  requested.
- Added requested-depth and completed-iteration tracking to the search result.
- Preserved the best move and score only after an iteration has been completed
  in full.
- Added a legal fallback root move before the first iteration begins.
- If a time limit interrupts an iteration, the engine now returns the best move
  from the last fully completed iteration instead of committing a partial
  result.
- The best move from the previous completed iteration is searched first at the
  root of the next iteration.
- Search statistics accumulate across all iterations executed by one go
  command.
- All iterations share the same search start time and time limit.
- Added final search reporting of the number of completed iterations compared
  with the requested number.
- Preserved a dedicated fixed-depth path for the diagnostic regression
  inventory, maintaining controlled comparison with alpha-002.
- Updated UCI identification to:
  id name PHILOCHESS 0.2-alpha-003

New internal diagnostic commands:
- idtest [depth]
- iterativetest [depth]

Both command names execute the same iterative-deepening test suite.

The suite runs controlled searches from the initial position and verifies for
each requested depth that:
- the engine has a legal best move;
- the search is not stopped;
- the requested depth is recorded correctly;
- every iteration from depth 1 through the requested depth is completed;
- the final completed depth equals the requested depth.

Version boundary:
Alpha-003 reports one final UCI search-information block and an iteration
summary. It does not yet emit a complete principal variation or a separate UCI
information line after every completed iteration.

Complete principal variation tracking and the fuller presentation of iterative
UCI reports belong mainly to alpha-004.

Important note:
Alpha-003 does not change the static evaluation function.
PHILOCHESS still uses material-only evaluation.

Iterative deepening reorganises the search and provides a stable completed
result under time pressure. By itself, it does not add positional knowledge or
guarantee a major immediate increase in playing strength.
