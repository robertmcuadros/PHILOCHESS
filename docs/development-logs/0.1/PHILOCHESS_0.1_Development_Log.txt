PHILOCHESS 0.1-alpha-015 — Game status detection

Implemented basic game status detection.

Technical milestone:
PHILOCHESS can now interpret the result of legal move generation and determine
the current game status for the side to move.

The engine now checks:
- which side is to move;
- whether that side is in check;
- how many legal moves are available;
- whether the position is normal, check, checkmate, or stalemate.

This version builds directly on alpha-014. The previous version already
filtered legal moves and could implicitly identify checkmate when the side to
move was in check and had no legal moves. Alpha-015 now gives that condition an
explicit chess meaning through a game status system.

New functionality:
- Added the GameStatus enum:
  Normal
  Check
  Checkmate
  Stalemate
- Added statusName().
- Added getGameStatus().
- Added printGameStatus().
- Added the internal debug command:
  status
- The status command now prints:
  side to move;
  number of legal moves;
  whether the side to move is in check;
  current game status.

Implemented basic alpha-beta pruning on top of the existing negamax structure.

Technical milestone:
PHILOCHESS can now search more efficiently by pruning branches that cannot
improve the final result.
This version keeps the basic negamax framework introduced in alpha-026 and the
useful UCI integration introduced in alpha-027, but replaces plain negamax
search with negamax plus alpha-beta pruning.

Alpha-028 builds directly on alpha-027.
The previous version improved the UCI command layer and added more practical
support for depth and time-based search commands.
Alpha-028 now improves the internal search efficiency while preserving the same
external UCI behavior.

New functionality:
- Added alpha-beta pruning to the existing negamax search.
- Replaced the plain negamax search function with an alpha-beta search
function.
- Added alpha and beta bounds to the recursive search.
- Added alpha-beta cutoff counting.

  go winc N binc N
  go movestogo N
  go infinite

- Preserved basic time-limit handling.
- Preserved material evaluation.
- Preserved eval.
- Preserved perft.
- Preserved divide.
- Preserved perftknown.
- Preserved the corrected Promotion bug trap reference case.
- Updated UCI identification to:
  id name PHILOCHESS 0.1-alpha-028

Important note:
Alpha-028 does not change the evaluation function.
The engine is still evaluating positions using material only.
The improvement is in search efficiency, not yet in positional understanding.

Another important note:
Alpha-beta pruning is more effective when good moves are searched first.
PHILOCHESS does not yet have move ordering, so the current alpha-beta
implementation is basic.
Even so, the node reduction already appears clearly in the tests.
