PHILOCHESS 0.1-alpha-007 — Estructura Move y lectura de movimientos UCI

Se implementó una estructura formal Move para representar internamente las jugadas.

Hito técnico:
PHILOCHESS ya puede convertir movimientos en formato UCI a datos internos y convertir esos datos internos nuevamente a texto UCI.
Cada jugada ahora almacena:
- casilla de origen;
- casilla de destino;
- pieza de promoción opcional.

Nueva funcionalidad:
- Se añadió la estructura Move.
- Se añadieron los campos internos:
  from
  to
  promotion
- Se añadió parseMove() para convertir movimientos UCI como e2e4 y e7e8q en datos Move.
- Se añadió soporte de moveToString() para objetos Move.
- Se añadió soporte para el campo promotion, usando '-' cuando no hay promoción.
- Se actualizó la aplicación de movimientos simples para trabajar con objetos Move.
- Se actualizó la generación de movimientos simples para devolver objetos Move.
- Se añadió el comando interno de depuración:
  move
- Se actualizó go para que PHILOCHESS devuelva la primera jugada generada desde su lista interna, en lugar de responder siempre la jugada fija e2e4.

Prueba realizada:
uci
isready
position startpos
move e2e4
move g1f3
move e7e8q
moves
go
quit

Resultado:
El comando move e2e4 devolvió:
e2e4 -> from 12 to 28 promotion -

El comando move g1f3 devolvió:
g1f3 -> from 6 to 21 promotion -

El comando move e7e8q devolvió:
e7e8q -> from 52 to 60 promotion q

El comando moves generó correctamente las 20 jugadas simples blancas desde la posición inicial.
La lista incluyó:
- 16 movimientos de peones.
- 4 movimientos de caballos.

La salida terminó con:
Total: 20

El comando go devolvió:
bestmove b1c3

Esto confirma que PHILOCHESS ya no depende de un bestmove fijo. Ahora puede elegir la primera jugada generada desde su lista interna de movimientos.

- La promoción ya está representada internamente, pero todavía no se valida completamente su legalidad.
- La generación de movimientos todavía está limitada a peones y caballos.
- Todavía no hay validación completa de legalidad.
- Todavía no hay validación de seguridad del rey.
- Todavía no hay detección de jaque.
- Todavía no hay enroque.
- Todavía no hay captura al paso.
- Todavía no hay generación de movimientos de alfiles.
- Todavía no hay generación de movimientos de torres.
- Todavía no hay generación de movimientos de dama.
- Todavía no hay generación de movimientos de rey.

Conclusión:
PHILOCHESS 0.1-alpha-007 queda completado.
El motor ahora tiene una representación interna más limpia de las jugadas y queda mejor preparado para la futura generación de movimientos, promociones, validación legal y búsqueda.
